v0.9.4 brings macOS Electron desktop support, hardened install scripts, and multiple CI/build fixes.

## 🍎 macOS Desktop Support

- **Server startup crash fixed** — `bundle-deps.cjs` had an overbroad skip pattern that dropped vendored runtime JS in `@fastify/busboy` (dicer), causing the Electron app to fail on launch
- **Universal native addons** — `better-sqlite3`, `sqlite-vec`, and `sharp` are now built as universal (arm64+x64) binaries for macOS, supporting both Apple Silicon and Intel Macs
- **App icon** — `electron-builder.yml` uses `icon.png` directly (auto-converted by electron-builder), `generate-icons.cjs` outputs Retina 512×512 with tray-icon properly written to disk

## 🛠️ Install Script Hardening

- **SSL error recovery** — detect SSL certificate errors during `pnpm install` and attempt source-compilation fallback for native modules
- **Git conflict handling** — `git pull` conflict auto-recovery with stash and re-pull
- **pnpm 11 compatibility** — dual-config fallback: `.npmrc` `onlyBuiltDependencies[]` for pnpm 11, `pnpm-workspace.yaml` for pnpm 10
- **Python detection** — enforce Python >= 3.8 requirement for node-gyp 12+
- **Version management** — recommend manual upgrade when Node.js/pnpm version is too old, auto-install only when completely missing

## 🐛 Build & CI Fixes

- **Workspace mode regression** — `pnpm-workspace.yaml` forced pnpm 10 into workspace mode, breaking subdirectory dependency resolution in CI and desktop-release workflows. Fixed both workflows and `bundle-deps.cjs` to handle workspace mode correctly
- **Missing Electron deps** — `bundle-deps.cjs` failed to discover `electron-store` and `electron-updater` in workspace mode, causing startup crash. Script now temporarily disables workspace mode during dependency scanning

## 🎨 UI

- **React 19 types** — added explicit React imports for `@types/react@19` compatibility
- **Language detection** — `resolveUILanguage` now prioritizes desktop config over `UI_LANGUAGE` env var

v0.9.4 带来 macOS Electron 桌面支持、安装脚本加固及多项 CI/构建修复。

## 🍎 macOS 桌面支持

- **服务端启动崩溃修复** — `bundle-deps.cjs` 中过于宽泛的跳过模式丢弃了 `@fastify/busboy`（dicer）的 vendor JS，导致 Electron 启动失败
- **Universal 原生模块** — `better-sqlite3`、`sqlite-vec`、`sharp` 现构建为 universal（arm64+x64）二进制，同时支持 Apple Silicon 和 Intel Mac
- **应用图标** — `electron-builder.yml` 直接使用 `icon.png`（由 electron-builder 自动转换），`generate-icons.cjs` 输出 Retina 512×512，tray-icon 正确写入磁盘

## 🛠️ 安装脚本加固

- **SSL 错误恢复** — 检测 `pnpm install` 期间的 SSL 证书错误，尝试通过源码编译回退方案安装原生模块
- **Git 冲突处理** — `git pull` 冲突自动 stash 后重新 pull 恢复
- **pnpm 11 兼容** — 双配置回退：`.npmrc` `onlyBuiltDependencies[]` 适配 pnpm 11，`pnpm-workspace.yaml` 适配 pnpm 10
- **Python 检测** — 强制要求 Python >= 3.8 以满足 node-gyp 12+
- **版本管理** — Node.js/pnpm 版本过旧时建议手动升级，仅在完全缺失时自动安装

## 🐛 构建与 CI 修复

- **Workspace 模式回归** — `pnpm-workspace.yaml` 使 pnpm 10 进入 workspace 模式，破坏了 CI 和 desktop-release 中的子目录依赖解析。修复了两个 workflow 和 `bundle-deps.cjs`
- **Electron 依赖缺失** — `bundle-deps.cjs` 在 workspace 模式下漏掉了 `electron-store` 和 `electron-updater`，导致启动崩溃。脚本现暂时禁用 workspace 模式进行依赖扫描

## 🎨 UI

- **React 19 类型** — 添加显式 React import 以兼容 `@types/react@19`
- **语言检测** — `resolveUILanguage` 现在优先使用桌面配置而非 `UI_LANGUAGE` 环境变量
