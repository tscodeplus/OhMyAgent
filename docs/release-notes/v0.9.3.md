v0.9.3 fixes macOS compatibility issues in the Electron desktop app, install script, and build pipeline.

## 🐛 Bug Fixes

- **macOS server startup crash** — `bundle-deps.cjs` had an overbroad skip pattern (`/^deps\//`) that dropped vendored runtime JS in `@fastify/busboy` (dicer module), causing the Electron app to fail on launch. Pattern tightened to only skip the expected platform-native packages.
- **macOS install script** — multiple fixes for `install.sh`: Node.js auto-install/upgrade when missing or too old, enforce pnpm >= 9 with auto-upgrade, fix BSD `grep -oP` incompatibility, handle SSL certificate errors, incompatible lockfile detection, and pnpm v10+ build script approval prompt
- **macOS app icon** — `electron-builder.yml` now uses `icon.png` directly (auto-converted by electron-builder) instead of a manually-created `icon.icns`, and `generate-icons.cjs` properly writes `tray-icon.png` to disk with 512×512 Retina output
- **pnpm v10+ workspace compatibility** — added `onlyBuiltDependencies` to `pnpm-workspace.yaml` so native modules build correctly under pnpm v10+

v0.9.3 修复了 macOS 下 Electron 桌面应用、安装脚本和构建流程的兼容性问题。

## 🐛 修复

- **macOS 服务端启动崩溃** — `bundle-deps.cjs` 中过于宽泛的跳过模式 (`/^deps\//`) 丢弃了 `@fastify/busboy`（dicer 模块）中 vendor 的运行时 JS，导致 Electron 应用启动失败。已收紧模式，仅跳过预期的平台原生包。
- **macOS 安装脚本** — `install.sh` 多项修复：Node.js 缺失或版本过旧时自动安装/升级、强制要求 pnpm >= 9 并自动升级、修复 BSD `grep -oP` 不兼容、处理 SSL 证书错误、检测不兼容的 lockfile、处理 pnpm v10+ 构建脚本审批提示
- **macOS 应用图标** — `electron-builder.yml` 改为直接使用 `icon.png`（由 electron-builder 自动转换），不再使用手动创建的 `icon.icns`；`generate-icons.cjs` 修复 `tray-icon.png` 未写入磁盘的问题，输出 512×512 Retina 分辨率
- **pnpm v10+ workspace 兼容** — `pnpm-workspace.yaml` 中添加 `onlyBuiltDependencies`，确保 pnpm v10+ 下原生模块正常编译
