v1.2.2 upgrades the embedded pi-mono from v0.80.7 to v0.80.10, spanning three upstream releases with new providers, dynamic tool loading, and model catalog refresh.

## 🧠 pi-mono v0.80.7 → v0.80.10

### New Providers
- **kimi-coding**, **moonshotai**, **opencode**, **xiaomi**, **zai**, **radius**, **xai** — 7 new provider integrations

### AI Layer
- **Dynamic tool loading** — tools can now be loaded on demand rather than upfront
- **xhigh/max thinking levels** — extended reasoning tiers for capable models
- **Model catalog refresh** — OpenRouter (320 lines changed) and other provider models updated
- **ModelRuntime compat layer** — `registerModel` wrapper, merged `getModel`/`getModels`/`getProviders` for unified model resolution

### OAuth Reorganization
- OAuth modules moved from `utils/oauth/` → `auth/oauth/` with backward-compat re-exports

### Breaking Changes Adapted
- `prepareNextTurn` → `prepareNextTurnWithContext` (upstream v0.80.8 breaking change) — adapted with unified signature
- `fallbackModels` added to `AgentOptions`, `compactToolsForPrompt` reintegrated in agent-loop

### Build Fixes
- `.ts` → `.js` imports across all new pi-mono files
- `BodyInit` type cast in `openai-codex-responses.ts`
- tsconfig/vitest paths for `@earendil-works/pi-ai/compat` subpath

Full upgrade record: `docs/PI_MONO_UPGRADE_v0807_to_v08010.md`

v1.2.2 将内嵌的 pi-mono 从 v0.80.7 升级至 v0.80.10，跨越三个上游版本，新增 7 个 provider、动态工具加载和模型目录刷新。

## 🧠 pi-mono v0.80.7 → v0.80.10

### 新增 Provider
- **kimi-coding**、**moonshotai**、**opencode**、**xiaomi**、**zai**、**radius**、**xai** — 7 个新 provider 集成

### AI 层
- **动态工具加载** — 工具支持按需加载而非全量预载
- **xhigh/max 思考级别** — 为高性能模型提供扩展推理层级
- **模型目录刷新** — 更新 OpenRouter（320 行变更）及其他 provider 模型数据
- **ModelRuntime 兼容层** — `registerModel` 包装器，合并 `getModel`/`getModels`/`getProviders` 统一模型解析

### OAuth 重组
- OAuth 模块从 `utils/oauth/` 迁移至 `auth/oauth/`，保留向后兼容的转导出

### 适配的破坏性变更
- `prepareNextTurn` → `prepareNextTurnWithContext`（上游 v0.80.8 破坏性变更），统一签名适配
- `fallbackModels` 添加到 `AgentOptions`，`compactToolsForPrompt` 重新集成到 agent-loop

### 构建修复
- 所有新增 pi-mono 文件的 `.ts` → `.js` 导入修正
- `openai-codex-responses.ts` 中 `BodyInit` 类型转换
- tsconfig/vitest 添加 `@earendil-works/pi-ai/compat` 子路径

完整升级记录：`docs/PI_MONO_UPGRADE_v0807_to_v08010.md`
