v0.6.0 introduces a redesigned Agent Team trigger system (P0–P4), session history restore on restart, file download, video sending, and policy mode enhancements.

## 🚀 New Features

### Redesigned Agent Team Trigger System

The Agent Team trigger logic has been upgraded from simple keyword matching to a four-tier progressive decision system:

- **P0 — Spawn gate + unified concurrency**: `spawn.enabled` switch controls tool visibility; `max_parallel` now respects `smart_agent_team.max_children` instead of hardcoded 3; legacy agents without spawn config auto-inherit the global setting
- **P2 — Judgment signals + Plan-Before-Spawn**: 5 positive signals (2 hits triggers spawn) + 4 negative signals (1 hit blocks it); mandatory `<plan>` tag before spawn calls
- **P3 — Turn counter + burst detection + reflection injection**: 3 trigger conditions — ≥6 tool calls in a single turn triggers immediate reflection, ≥8 serial calls without spawn prompts a suggestion, ≥5 idle turns sends a gentle reminder; injected via message append so the system prompt cache stays intact
- **P4 — plan_and_spawn structured tool + DAG executor**: topological sort, cycle detection, chunked parallel execution; sequential strategy with failure blocking; registered in the advanced tool profile

### Session History Restore on Restart

After server restart, recent conversation history is automatically loaded from the database so ongoing sessions continue without interruption.

### File Download

A new `download_file` tool and public download endpoint allow agents to fetch external files on demand during conversations.

### Video Sending

`_send_media` across all channels now supports video: auto thumbnail extraction, video duration metadata, and support for Feishu / Telegram / WeChat / QQ.

## 🛠 Improvements

- **Policy mode enhancements** — new bypass mode (auto-approve all tool calls), WebUI mode dropdown selector, `/permission` command triggers config reload across all channels
- **Plan tag rendering** — `<plan>` blocks rendered as markdown blockquotes, XML tags stripped while preserving inner markdown content
- **Feishu card resilience** — `cardid-invalid` errors auto-recover by recreating the card and falling back to text reply

## 🚀 新功能

### Agent Team 触发机制重新设计

从简单关键词匹配升级为四层递进式智能决策：

- **P0 — spawn 门控与统一并发**：`spawn.enabled` 开关控制工具可见性；`max_parallel` 统一为 `max_children` 配置；旧 agent 自动继承全局开关
- **P2 — 判断信号 + Plan-Before-Spawn**：5 个正向信号（2 命中触发）+ 4 个负向信号（1 命中阻止）；强制 spawn 前输出 `<plan>` 规划
- **P3 — 轮次计数 + 爆发检测 + 反思注入**：单轮 ≥6 次工具调用→立即反思、≥8 次串行无 spawn→提示、≥5 轮空闲→温和提醒；追加消息注入不影响缓存
- **P4 — plan_and_spawn 工具 + DAG 执行器**：拓扑排序、循环检测、分块并行执行；串行策略失败阻断；已注册到 advanced 工具集

### 重启服务历史会话加载

服务重启后自动从数据库恢复近期对话历史，继续之前的会话不中断。

### 文件下载

新增 `download_file` 工具和公开下载端点，Agent 可按需下载外部文件。

### 视频发送

全频道 `_send_media` 支持视频：自动缩略图、视频时长、Feishu / Telegram / WeChat / QQ 全覆盖。

## 🛠 改进

- **策略模式增强** — 新增 bypass 模式（无审批自动执行）、WebUI 下拉选择器、`/permission` 全频道热重载
- **Plan 标签渲染** — `<plan>` 标签以引用块样式展示，剥离 XML 保留 Markdown
- **飞书卡片容错** — `cardid-invalid` 自动重建卡片并降级文本回复
