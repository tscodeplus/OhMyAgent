v0.6.0 introduces a redesigned Agent Team trigger system (P0–P4), session history restore on restart, file download, video sending, and policy mode enhancements.

## 🚀 New Features

### Agent Team 触发机制重新设计

Agent Team 的触发逻辑从简单关键词匹配升级为四层递进式智能决策：

- **P0 — spawn 门控与统一并发**：`spawn.enabled` 开关控制工具可见性；`max_parallel` 统一为 `max_children` 配置；旧 agent 自动继承全局开关
- **P2 — 判断信号 + Plan-Before-Spawn**：5 个正向信号（满足 2 触发）+ 4 个负向信号（命中 1 阻止）；强制 spawn 前输出 `<plan>` 规划
- **P3 — 轮次计数 + 爆发检测 + 反思注入**：单轮 ≥6 次工具调用立即反思、≥8 次串行无 spawn 提示、≥5 轮空闲温和提醒；追加消息注入不影响缓存
- **P4 — plan_and_spawn 工具 + DAG 执行器**：拓扑排序、循环检测、分块并行执行；串行策略失败阻断；已注册到 advanced 工具集

### 重启服务历史会话加载

服务重启后自动从数据库恢复近期对话历史，继续之前的会话不中断。

### 文件下载

新增 `download_file` 工具和公开下载端点，Agent 可在对话中按需下载外部文件。

### 视频发送

所有频道的 `_send_media` 工具现已支持视频：自动缩略图提取、视频时长信息、全频道（Feishu / Telegram / WeChat / QQ 等）。

## 🛠 Improvements

- **Plan 标签渲染** — `<plan>` 标签以引用块样式展示，剥离 XML 保留内部 Markdown，自动分隔后续工具调用
- **策略模式增强** — 新增 bypass 模式（无审批自动执行），WebUI 模式下拉选择器，`/permission` 命令全频道配置热重载
- **飞书卡片容错** — `cardid-invalid` 时自动重建卡片并降级为文本回复
