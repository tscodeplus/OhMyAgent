v0.9.0 upgrades the embedded pi-mono framework from v0.79.6 to v0.80.2 and fixes several persona, skill loader, and CI issues.

## 🚀 New Features

### pi-mono Framework Upgrade (v0.79.6 → v0.80.2)

The embedded pi-mono multi-provider AI agent framework has been upgraded across 7 upstream releases, bringing:

- **AI layer restructuring** — providers organized into `api/`, new `auth/` subsystem, new factory-based provider system with backward-compatible `compat.ts` entrypoint
- **Post-compaction token estimates** — more accurate token counting after context compaction
- **Mistral prompt caching** — reduced latency and cost for repeat system prompts via `x-affinity` header
- **Chat-template thinking** — native thinking/reasoning support for vLLM, DeepSeek, and other OpenAI-compatible providers using chat template injection
- **Extension compaction events** — extensions can now respond to `reason` and `willRetry` compaction lifecycle events
- **Provider fixes** — Bedrock, Cloudflare, Fireworks, Together AI compatibility improvements
- All local customizations preserved: fallbackModels, ohmyagent_agentName, deferred tools, registerModel, dynamic model resolution, ProviderEnv, cacheWrite1h, graceful OAuth

## 🐛 Bug Fixes

- **Stale name-preference memories** — memories containing preferred-name expressions no longer override the persona system; persona header strengthened to take priority over all memory sources
- **Skill loader dot-directory noise** — hidden directories (`.pi/`) under `skills/` are now excluded from skill scanning, eliminating spurious "[skill-loader] Skipping skill" warnings
- **pnpm 10 CI compatibility** — restored `pnpm.onlyBuiltDependencies` in `package.json` so native modules compile correctly in CI
- **TypeScript type mismatch** — widened `getModel` type parameter so custom provider names no longer cause compile errors
- **Flaky circuit breaker test** — replaced `setTimeout`-based timing tests with deterministic `nowFn` injection

## 🚀 新功能

### pi-mono 框架升级（v0.79.6 → v0.80.2）

嵌入式 pi-mono 多 Provider AI Agent 框架跨越 7 个上游版本升级：

- **AI 层重构** — Provider 按目录重组为 `api/`，新增 `auth/` 子系统，基于工厂模式的 Provider 注册系统，通过 `compat.ts` 向后兼容
- **Compaction 后 Token 估算** — 上下文压缩后更精确的 Token 计数
- **Mistral Prompt 缓存** — 通过 `x-affinity` header 减少重复系统提示词的延迟和费用
- **Chat-template thinking** — vLLM、DeepSeek 及其他 OpenAI 兼容 Provider 的原生思考/推理支持
- **扩展 Compaction 事件** — 扩展可响应 `reason` 和 `willRetry` 等 compaction 生命周期事件
- **Provider 修复** — Bedrock、Cloudflare、Fireworks、Together AI 兼容性改进
- 所有本地定制保留：fallbackModels、ohmyagent_agentName、deferred tools、registerModel、动态模型解析、ProviderEnv、cacheWrite1h、graceful OAuth

## 🐛 修复

- **过时名称偏好记忆** — 含 preferred-name 表达的记忆不再覆盖 Persona 系统；Persona header 优先级提升至高于所有记忆来源
- **技能加载器隐藏目录警告** — `skills/` 下的隐藏目录（`.pi/`）不再被扫描，消除无意义的 "[skill-loader] Skipping skill" 警告
- **pnpm 10 CI 兼容** — 恢复 `package.json` 中的 `pnpm.onlyBuiltDependencies`，确保 CI 中正确编译原生模块
- **TypeScript 类型不匹配** — 拓宽 `getModel` 类型参数，自定义 Provider 名称不再导致编译错误
- **电路断路器测试不稳定** — 将 `setTimeout` 时序测试替换为确定性的 `nowFn` 注入
