## 🛠 Fixes & Improvements

### Approval System

- **Auto-reject pending approvals on steer** — When a user sends a new message while approvals are pending, those approvals are now automatically rejected before the new message is processed. This prevents orphaned approval cards and ensures the conversation state stays consistent
- **Correct steer ordering** — Steer messages are now queued before resolving pending approvals, so the interruption appears in the correct timeline order
- **Fixed model label in interrupted footer** — Messages interrupted by steer now show the correct model name in their footer instead of stale data

### Approval Cards

- **Real-time card update after steer** — The WebUI approval card now updates immediately when a steer/replace message is sent, reflecting the new tool execution state
- **Tool indicator preserved** — After steer, the tool call indicator in the message list is correctly maintained

### Channel Approvals

- **WeChat approval result updates** — `updateApprovalResult` now properly implemented for WeChat, so approval cards reflect the resolved state after decisions
- **QQ approval result updates** — Same fix for QQ channel approval sender
- **Feishu reject-before-swap** — Pending approvals are now rejected before swapping the approval card, preventing stale approval states

### Orchestrator

- **Config defaults now applied** — When the `orchestrator` section was missing entirely from `config.yaml`, default values (`enabled: true`, `maxChildAgents: 4`, etc.) were not being applied. Fixed.

### WebUI

- **Debug logging** — Added debug-level logging to the steer route for easier troubleshooting of approval workflows

## 🛠 修复与改进

### 审批系统

- **Steer 时自动拒绝待处理审批** — 用户在有待处理审批时发送新消息，系统会自动拒绝这些审批后才处理新消息，避免审批卡片悬挂和对话状态不一致
- **修正 steer 操作顺序** — 用户进行分支/替换操作时，消息先入队再用新消息标记拒绝待处理审批，确保时间线正确
- **修正中断消息的模型名称** — 被 steer 中断的消息现在在 footer 中显示正确的模型名称

### 审批卡片

- **Steer 后实时更新卡片** — WebUI 审批卡片在发送分支/替换消息后立即更新，反映新的工具执行状态
- **保留工具调用指示器** — Steer 后消息列表中的工具调用指示器正确维护

### 渠道审批

- **微信审批结果更新** — `updateApprovalResult` 现在在微信渠道正确实现，审批卡片在决策后反映最新状态
- **QQ 审批结果更新** — QQ 渠道同样的修复
- **飞书先拒绝后替换** — 在替换审批卡片之前先拒绝待处理审批，防止残留审批状态

### 编排器

- **配置默认值修复** — 当 `config.yaml` 中完全缺失 `orchestrator` 配置段时，默认值（`enabled: true`, `maxChildAgents: 4` 等）未正确应用的问题已修复

### WebUI

- **调试日志** — 在 steer 路由中增加调试级别日志，便于排查审批流程问题
