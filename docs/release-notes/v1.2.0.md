## 📝 Feishu Markdown Rendering Overhaul

A series of fixes addressing long-standing Feishu markdown rendering issues:

- **CJK-aware formatting** — rewritten `fixFeishuBold` → `fixFeishuMarkdown` with two-pass ZWSP (zero-width space) insertion for `**bold**`, `*italic*`, `~~strikethrough~~` when adjacent to CJK/fullwidth characters, preventing Feishu from eating markdown markers next to Chinese/Japanese/Korean text
- **HTML tag stripping** — added `stripHtmlTags` to remove `<details>`, `<summary>` and other HTML elements the model may output, preserving inner text content (Feishu `lark_md` does not support HTML)
- **Table alignment fix** — added `fixTableAlignment` to correct double-colon alignment syntax (`: :---:`) that LLMs sometimes produce, which breaks Feishu table rendering
- **Plan tag partial buffer** — fixed `TAG_STARTS` typo (`</pla` → `</plan`) in `filterPlanDelta` to correctly buffer partial `</plan>` tags split across streaming deltas

## 💬 Command Reply Rendering

- **CardKit 2.0 replies** — switched command replies from `msg_type:text` (no markdown support) to `msg_type:interactive` with CardKit 2.0 cards via `buildSimpleMarkdownCard`, so `**bold**` and other formatting now render correctly in bot responses
- **i18n plan labels** — added `plan.label` keys for en and zh-CN; raw `<plan>` / `</plan>` tags replaced with localized 📋 headers across all channels (Feishu, Telegram, WeChat, QQ)

## 📊 Test Coverage

## 📝 飞书 Markdown 渲染全面改造

一系列修复解决了长期存在的飞书 Markdown 渲染问题：

- **CJK 字符感知格式化** — 将 `fixFeishuBold` 重写为 `fixFeishuMarkdown`，采用双遍 ZWSP（零宽空格）插入，处理 CJK/全角字符相邻的 `**粗体**`、`*斜体*`、`~~删除线~~`，防止飞书在中文/日文/韩文旁边吞掉 Markdown 标记
- **HTML 标签剥离** — 新增 `stripHtmlTags`，移除模型可能输出的 `<details>`、`<summary>` 等 HTML 元素，保留内部文本内容（飞书 `lark_md` 不支持 HTML）
- **表格对齐修复** — 新增 `fixTableAlignment`，纠正 LLM 偶尔产生的双冒号对齐语法（`: :---:`），该语法会破坏飞书表格渲染
- **Plan 标签部分缓冲** — 修复 `filterPlanDelta` 中的 `TAG_STARTS` 拼写错误（`</pla` → `</plan>`），正确处理跨流式 delta 分割的部分 `</plan>` 标签

## 💬 命令回复渲染

- **CardKit 2.0 回复** — 将命令回复从 `msg_type:text`（不支持 Markdown）切换为 `msg_type:interactive`，使用 CardKit 2.0 卡片通过 `buildSimpleMarkdownCard` 渲染，使 `**粗体**` 等格式在机器人回复中正确显示
- **i18n Plan 标签** — 新增英文和中文的 `plan.label` 键值；原始 `<plan>` / `</plan>` 标签替换为本地化的 📋 标题，覆盖所有渠道（飞书、Telegram、微信、QQ）
