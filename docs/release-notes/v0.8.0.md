v0.8.0 is a major release bringing QR code auto-configuration for all channels, custom bot names, and unified desktop i18n.

## 🚀 New Features

### QR Code Auto-Configuration for All Channels

Set up messaging channels by scanning a QR code — no manual credential entry needed:

- **Feishu / Lark** — device authorization flow (OAuth device_code): scan QR with Feishu or Lark app to auto-create and register the bot; region selector (domestic 飞书 vs international Lark)
- **WeChat** — integrates existing iLink QR login flow via `/api/channels/wechat/qr*`
- **QQ** — inline credential input flow (scan QR to capture App ID + secret)
- **Telegram** — BotFather deep-link QR with inline token input for one-shot setup

**QRCodeModal** — shared frontend component with full state machine: loading → QR display → polling → confirmed → expired/error. Includes LAN IP detection, `QrSessionStore` for server-side session management, and reconfirmation overlay for already-configured channels.

### Custom Bot Name for Feishu and Telegram

- **Feishu** — `botName` is passed as `app_name` during QR registration to pre-fill the bot name at creation time (matches node-sdk v1.67.0 `appPreset` mechanism)
- **Telegram** — `botName` triggers `setMyName` API call on startup
- Config schema extended with `botName` field for both channels
- UI input in channels settings with i18n placeholder ("Leave empty to use default")

## 🔧 Refactoring

### Unified Desktop i18n

Electron main process i18n has been unified to shared locale JSON files:

- New `desktop.json` locale files under `src/locales/{en,zh-CN}/` with 5 sections: updater, tray, gateway, splash, error
- Lightweight i18n loader (`desktop/src/i18n.ts`) using `readFileSync` + `JSON.parse`
- Removed ~46 lines of hardcoded Chinese from updater, tray, and main process
- Tray menu rebuilds instantly on language change (no restart needed)
- Language preference persisted to `electron-store` for restart survival
- WebUI language changes synced to Electron main process via IPC

## 🐛 Bug Fixes

- **Channel config YAML round-trip** — WebUI saves config with camelCase keys but the YAML loader only read snake_case. Now both formats are accepted via nullish coalescing so saved credentials survive a config reload.
- **Feishu QR registration** — fixed multiple param mismatches with Feishu API: switched to `form-urlencoded` (was JSON), changed auth method to `device_code`, aligned params with `@larksuiteoapi/node-sdk v1.67.0`
- **Desktop channel settings** — fixed conditional rendering using raw config instead of dirty state (QR buttons and manual inputs now appear when channel is toggled on); overwrite confirmation only shows when existing credentials are detected
- **QQ QR flow** — changed from local HTML exchange page to inline credential input, avoiding LAN IP reachability issues on desktop/WSL

v0.8.0 是一个重大版本更新：全渠道 QR 码自动配置、自定义 Bot 名称、桌面端国际化统一。

## 🚀 新功能

### 全渠道 QR 码自动配置

扫码即可配置消息渠道，无需手动输入凭据：

- **飞书 / Lark** — 设备授权流程（OAuth device_code）：用飞书或 Lark App 扫码自动创建并注册机器人；支持国内飞书 / 国际 Lark 区域选择
- **微信** — 集成现有 iLink 扫码登录，通过 `/api/channels/wechat/qr*` 接口
- **QQ** — 内联凭据输入流程（扫码获取 App ID + Secret）
- **Telegram** — BotFather 深度链接二维码 + 内联 Token 输入，一步完成配置

**QRCodeModal** — 共享前端组件，覆盖完整状态机：加载 → 二维码展示 → 轮询 → 确认 → 过期/错误。包含局域网 IP 检测、`QrSessionStore` 服务端会话管理、已配置渠道的覆盖确认弹窗。

### 飞书和 Telegram 自定义 Bot 名称

- **飞书** — `botName` 在 QR 注册时作为 `app_name` 传入，创建时预填机器人名称（对齐 node-sdk v1.67.0 `appPreset` 机制）
- **Telegram** — `botName` 在启动时调用 `setMyName` API 设置
- 配置 schema 新增 `botName` 字段
- 渠道设置页面提供输入框，支持中英文占位提示（"留空使用默认名称"）

## 🔧 重构

### 桌面端国际化统一

Electron 主进程国际化已统一到共享 locale JSON 文件：

- 新增 `desktop.json` 语言文件（`src/locales/{en,zh-CN}/`），含 5 个模块：updater、tray、gateway、splash、error
- 轻量 i18n 加载器（`desktop/src/i18n.ts`），使用 `readFileSync` + `JSON.parse`
- 移除 updater、tray、main 进程中约 46 行硬编码中文
- 托盘菜单切换语言即时生效（无需重启）
- 语言偏好持久化到 `electron-store`，重启后保留
- WebUI 语言变更通过 IPC 同步到 Electron 主进程

## 🐛 修复

- **渠道配置 YAML 读写闭环** — WebUI 以 camelCase 保存配置，但 YAML 加载器只读 snake_case，导致保存后刷新凭据消失。现已改为两种格式均接受（nullish coalescing）
- **飞书 QR 注册** — 修复多项与飞书 API 的参数不匹配：切换为 `form-urlencoded`（原为 JSON）、认证方式改为 `device_code`、参数对齐 `@larksuiteoapi/node-sdk v1.67.0`
- **桌面端渠道设置** — 修复条件渲染使用 raw config 而非 dirty state（开启渠道后 QR 按钮和手动输入框正常显示）；仅在有凭据时才弹出覆盖确认
- **QQ QR 流程** — 从本地 HTML 交换页改为内联凭据输入，避免桌面/WSL 环境下局域网 IP 不可达问题
