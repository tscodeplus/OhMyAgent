v1.2.1 upgrades the embedded pi-mono from v0.80.6 to v0.80.7 and fixes thinking indicator animation and table alignment issues.

## 🧠 pi-mono v0.80.6 → v0.80.7

- **Session affinity format** — migrated from `sendSessionIdHeader` to `sessionAffinityFormat` for provider-level session routing (breaking change from upstream)
- **pi-messages API** — new `pi-messages.ts` (436 lines) for unified multi-provider messaging
- **Anthropic Messages API updates** — updated for latest Anthropic API changes (179 lines changed)
- **OAuth radius module** — new `oauth/radius.ts` (557 lines) for OAuth device-code flow enhancements
- **Model catalog updates** — OpenRouter and Vercel AI Gateway model lists refreshed

Full upgrade record: `docs/PI_MONO_UPGRADE_v0806_to_v0807.md`

## 🎨 Thinking Indicator Animation

- **WebUI** — CSS keyframe animation: 1→2→3→0 dots cycling at 2.4s period (0.6s/phase), replacing the previous static dots
- **Feishu** — setTimeout-based dot cycling at 600ms, matching WebUI timing
- **Auto-scroll** — MessageList now auto-scrolls to bottom when thinking indicator appears

## 🛠 Fixes

- **Table alignment** — additional fix for double-colon syntax (`|::---|` → `|:---|`) that some LLMs produce
- **Import extension** — `.ts` → `.js` in `images.ts` for Node.js runtime compatibility
