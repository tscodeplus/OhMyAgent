v0.5.1 brings richer custom provider model configuration, a redesigned About page with update checker, and CI stability fixes.

## 🚀 New Features

### About Page & Update Checker

The Desktop Settings tab has been replaced with a proper **About** page (visible in both WebUI and Electron):

- **App branding** — centered logo on a purple gradient, version number, and the "Remembers. Understands. Respects." slogan
- **Update checker** — "Check for Updates" button queries the GitHub Releases API and notifies you when a new version is available
- **Electron auto-update** — wired up IPC handlers for check, download, and install actions
- **GitHub links** — repository link and "Submit Issues" button for easy feedback
- **Friendly error handling** — GitHub unreachable or rate-limited? You get a clear message instead of a cryptic error

### Custom Provider Model Config

Custom provider models now support richer configuration:

- **Input modalities** — toggle text / image / video checkboxes for each custom model, so the system knows what the model can accept
- **maxTokens** — per-model token limit override
- **reasoningLevel** — set the reasoning effort level for models that support it
- **Fine-grained restart prompts** — only non-reasoningLevel changes trigger the "restart required" warning

## 🛠 Improvements

- **CI stability** — synced vitest and @vitest/coverage-v8 to matching 3.2.6 versions, eliminating "mixed versions" warnings; lowered coverage thresholds slightly to account for Ubuntu CI environment variance
- **i18n** — Chinese model provider section renamed to "模型供应商", new i18n keys for About page and model input

## 🚀 新功能

### 关于页面 & 更新检查

桌面设置页已替换为正式的**关于**页面（WebUI 和 Electron 均可看到）：

- **应用品牌** — 紫色渐变上的居中 Logo、版本号和"长于记忆 · 精于理解 · 忠于边界"口号
- **更新检查** — "检查更新"按钮查询 GitHub Releases API，有新版本时通知你
- **Electron 自动更新** — 接入 IPC 处理器，支持检查、下载和安装操作
- **GitHub 链接** — 仓库链接和"提交反馈"按钮，方便反馈
- **友好的错误处理** — GitHub 不可达或被限速？显示清晰提示而非晦涩错误

### 自定义模型配置增强

自定义供应商模型现在支持更丰富的配置：

- **输入模态** — 为每个自定义模型勾选 text / image / video，系统了解模型可接受的输入类型
- **maxTokens** — 单模型 token 上限覆盖
- **reasoningLevel** — 为支持推理的模型设置推理强度级别
- **精细重启提示** — 仅非 reasoningLevel 的变更才触发"需要重启"警告

## 🛠 改进

- **CI 稳定性** — 同步 vitest 和 @vitest/coverage-v8 至匹配的 3.2.6 版本，消除"mixed versions"警告；略微下调覆盖率阈值以适配 Ubuntu CI 环境的计算差异
- **i18n** — 中文供应商区域更名为"模型供应商"，为关于页面和模型输入新增 i18n keys
