## 🛠 Fixes & Improvements

### One-Click Install

- **Windows terminal mojibake fixed** — Replaced all Unicode box-drawing and special characters (`╔╗║✓✗→`) with ASCII equivalents so install output renders correctly regardless of system code page
- **Missing UI dependencies** — WebUI dependency install and build steps (`ui/pnpm install && pnpm build`) added to the install scripts; no more Vite errors about missing React, react-router-dom, etc. on fresh installs
- **Admin privilege check** — Service installation step detects non-elevated PowerShell and provides clear instructions instead of failing silently
- **Updated default models** — DeepSeek defaults: `deepseek-v4-flash` (model) and `deepseek-v4-pro` (reasoning)
- **Skip completed config** — If `.env` already has an API key + WebUI token, the interactive provider selection is skipped
- **Config consolidation** — Provider settings are written to `config.yaml` (`provider` + `provider_keys` sections); `.env` only holds `WEBUI_TOKEN` + `LOG_LEVEL`. No more duplication, and the Settings UI shows editable API key fields

### Dependencies

- **sqlite-vec made mandatory** — Platform-specific packages (`sqlite-vec-windows-x64`, `sqlite-vec-linux-x64`, etc.) are now direct dependencies instead of optional. A missing native extension will fail the install rather than silently falling back to cosine similarity
- **Eager startup probe** — sqlite-vec is loaded immediately at startup with detailed error logging (DLL missing, platform mismatch, etc.) instead of a generic "unavailable" warning

### Channels

- **Feishu disabled by default** — Fresh installs no longer attempt to connect to Lark with placeholder credentials, eliminating `code: 1000040346, system busy` and `PingInterval` errors
- **Guarded FeishuClient creation** — The Lark SDK client is only instantiated when Feishu is both enabled and configured with real app credentials

### WebUI

- **Idempotent default project** — New `POST /api/projects/ensure-default` endpoint uses a database transaction to atomically create a default space. Multiple browser tabs can no longer race to create duplicates
- **App-startup creation** — Default project is ensured by `ProjectProvider` on mount, so visiting `/dashboard` or any route directly still triggers auto-creation
- **I18n project name** — `Default Space` / `默认空间` is now localized via the frontend instead of being hardcoded
- **Session list refresh** — The session sidebar updates immediately after creating a new session via the "+" button; no manual refresh needed

## 🛠 修复与改进

### 一键安装

- **Windows 终端乱码修复** — 所有 Unicode box-drawing 和特殊字符替换为 ASCII 等效字符，安装输出在任何系统 code page 下都正常显示
- **缺失的 UI 依赖** — 安装步骤中新增 WebUI 依赖安装和构建，不再报 react、react-router-dom 等缺失错误
- **管理员权限检查** — 服务安装前检测非管理员 PowerShell 并给出清晰指引，不会静默失败
- **默认模型更新** — DeepSeek 默认模型改为 `deepseek-v4-flash`，推理模型改为 `deepseek-v4-pro`
- **跳过已完成配置** — `.env` 中已有 API Key + WebUI Token 时跳过交互式 provider 选择
- **配置不再重复** — Provider 配置统一写入 `config.yaml`（`provider` + `provider_keys`），`.env` 只保留 `WEBUI_TOKEN` 和 `LOG_LEVEL`。Settings UI 中 API Key 可正常编辑

### 依赖

- **sqlite-vec 强制依赖** — 平台特定包改为直接依赖，原生扩展缺失时安装会报错，不再静默降级为 cosine 相似度
- **启动探针** — 启动时主动加载 sqlite-vec 并输出详细错误信息（DLL 缺失、平台不匹配等），不再只显示 generic "unavailable" 警告

### 渠道

- **飞书默认禁用** — 新安装不再以占位凭据连接 Lark 服务器，消除 `code: 1000040346, system busy` 和 `PingInterval` 错误
- **FeishuClient 条件创建** — 仅在飞书启用且配置了真实凭据时才创建 lark SDK 客户端

### 管理面板

- **默认空间幂等创建** — 新增后端原子端点确保默认空间唯一创建，多标签页并发不会重复
- **应用启动即创建** — 默认空间逻辑提升到 `ProjectProvider`，直接访问 `/dashboard` 等任意路由也能触发自动创建
- **空间名称国际化** — `Default Space` / `默认空间` 使用前端 t() 传入，不再硬编码
- **会话列表实时刷新** — 新建会话后侧边栏立即更新，无需手动刷新
