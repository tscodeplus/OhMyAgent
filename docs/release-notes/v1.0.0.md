v1.0.0 marks the first stable release of OhMyAgent, featuring polished WebUI/desktop UX, improved message rendering, and robust cross-platform install scripts.

## 🎨 WebUI & Desktop UX

- **Thinking indicator** — animated "Thinking..." bubble now appears in the message list (styled as an assistant message), with a reliable CSS opacity-based dot animation replacing the broken `content` trick
- **Theme sync** — Electron nativeTheme title bar now automatically follows the WebUI dark/light mode setting, giving the desktop app a cohesive look across window chrome and content
- **Layout polish** — reduced chat area padding for tighter edge spacing, consistent spacing between top and bottom edges of the chat box
- **Message duplication fix** — restored `onStreamStart` call in `sendMessage` to prevent a `streamGenerationRef` race condition that caused duplicate messages after refetch

## 📝 Message Rendering

- **Line breaks preserved** — user messages now render as plain text instead of ReactMarkdown, so line breaks and whitespace are displayed exactly as typed
- **`<plan>` tags rendered** — AI `<plan>` XML tags are now converted to markdown blockquotes, making plan content visible and readable instead of being hidden

## 🏗️ Desktop Build

- **Workspace-hoisted deps backfill** — `bundle-deps.cjs` now detects packages hoisted to the root `node_modules` by workspace mode and copies them into the staging directory, fixing missing dependency errors in packaged builds
- **SkipRootBuild compatibility** — the build script now ensures `dist/package.json` exists even when `-SkipRootBuild` is used, preventing ESM resolution failures
- **Windows Start Menu** — added NSIS `menuCategory` for proper organization in the Windows Start Menu
- **Title bar safety** — `setTitleBarOverlay` is now guarded with try-catch to prevent crashes on systems where the API is unavailable

## 🛠️ Install Script Improvements

- **Proxy auto-detection** — `install.sh` now detects local proxy environments and automatically sets `https_proxy` for pnpm, fixing installation failures behind corporate proxies
- **Deploy script robustness** — `deploy-windows.sh` no longer fails when the scheduled task is not installed, allowing first-time deployments to succeed

***

v1.0.0 是 OhMyAgent 的首个稳定版本，带来打磨后的 WebUI/桌面端体验、改进的消息渲染和健壮的跨平台安装脚本。

## 🎨 WebUI 与桌面端体验

- **思考动画指示器** — 消息列表中新增动画 "Thinking..." 气泡（以助手消息样式呈现），使用可靠的 CSS 透明度动画替代了有问题的 `content` 技巧
- **主题同步** — Electron 原生标题栏现在自动跟随 WebUI 深色/浅色模式设置，桌面应用的窗口外观与内容风格保持一致
- **布局打磨** — 缩小聊天区域边距，输入框上下间距对称一致
- **消息重复修复** — 恢复了 `sendMessage` 中的 `onStreamStart` 调用，防止 `streamGenerationRef` 竞态条件导致重新加载后出现重复消息

## 📝 消息渲染

- **保留换行符** — 用户消息现在以纯文本渲染而非 ReactMarkdown，换行和空格将原样显示
- **`<plan>` 标签可见** — AI 的 `<plan>` XML 标签现转换为 Markdown 引用块，使计划内容可见可读而非被隐藏

## 🏗️ 桌面端构建

- **Workspace 提升依赖回填** — `bundle-deps.cjs` 现在检测被 workspace 模式提升至根 `node_modules` 的包，并将它们复制到打包目录，修复打包后缺失依赖的错误
- **SkipRootBuild 兼容** — 构建脚本现在确保即使使用 `-SkipRootBuild` 时 `dist/package.json` 也存在，防止 ESM 解析失败
- **Windows 开始菜单** — 添加 NSIS `menuCategory`，在 Windows 开始菜单中正确归类
- **标题栏安全保护** — `setTitleBarOverlay` 现在使用 try-catch 保护，防止在不支持该 API 的系统上崩溃

## 🛠️ 安装脚本改进

- **代理自动检测** — `install.sh` 现在能检测本地代理环境并自动为 pnpm 设置 `https_proxy`，解决企业代理背后的安装失败问题
- **部署脚本健壮性** — `deploy-windows.sh` 在计划任务未安装时不再失败，使首次部署能够成功
