v0.4.1 brings multimodal image-to-image and image-to-video generation, plus a major chat UX upgrade with drag-and-drop file uploads, inline media rendering, and per-session input memory.

## 🚀 New Features

### Image-to-Image & Image-to-Video Generation

The `image_generate` and `video_generate` tools now accept a `referenceImages` parameter — upload one or more reference images and the model will use them as style/ content guides. Provider-specific mapping is handled automatically (Agnes, ByteDance, and OpenAI-compatible providers all supported).

### WebUI File Attachments

- **Drag-and-drop upload** — drop images and files directly onto the chat input, or click the new attachment button
- **Parallel uploads** — multiple files upload simultaneously with individual progress indicators
- **Inline media display** — images from `webui_send_media` actions render at the tool call position in the message bubble, not pushed to the bottom
- **Download button** — every inline image and file link now has a download button with the real filename
- **Markdown rendering** — user messages containing images and links now render inline with ReactMarkdown
- **Vision model integration** — attached images are auto-extracted as multimodal `ImageContent` for vision-capable models

## 🛠 Improvements

- **Per-session input persistence** — chat input text and file uploads are preserved when switching between conversations; no more losing your draft
- **Model name display fix** — message footer now shows the full model name including provider prefix
- **Native image mode** — image handling prefers native multimodal over the vision bridge for vision-capable models, reducing unnecessary conversions
- **Upload size** — WebUI and desktop app file upload limit increased from 1MB to 50MB
- **Skills updated** — `image-gen` and `video-gen` skills now include img2img / img2vid guidance with trigger patterns, usage rules, and examples

## 🚀 新功能

### 图生图 & 图生视频

`image_generate` 和 `video_generate` 工具现在支持 `referenceImages` 参数——上传一张或多张参考图片，模型会将其作为风格/内容指导。供应商特定映射自动处理（Agnes、ByteDance、OpenAI 兼容供应商均已支持）。

### WebUI 文件附件

- **拖拽上传** — 直接将图片和文件拖入聊天输入框，或点击新增的附件按钮
- **并行上传** — 多个文件同时上传，各自显示进度
- **内联媒体显示** — `webui_send_media` 的图片在消息气泡中的工具调用位置渲染，不再堆在底部
- **下载按钮** — 每个内联图片和文件链接都有带真实文件名的下载按钮
- **Markdown 渲染** — 包含图片和链接的用户消息现在用 ReactMarkdown 内联渲染
- **视觉模型集成** — 附件图片自动提取为多模态 `ImageContent`，供视觉模型使用

## 🛠 改进

- **会话级输入记忆** — 切换对话时保留聊天输入文本和文件上传，不再丢失草稿
- **模型名称显示修复** — 消息底部现在显示包含供应商前缀的完整模型名称
- **原生图片模式** — 对支持视觉的模型优先使用原生多模态而非视觉桥接，减少不必要的转换
- **上传大小** — WebUI 和桌面端文件上传大小限制提升至 50MB
- **技能更新** — `image-gen` 和 `video-gen` 技能新增 img2img / img2vid 指导，含触发模式、使用规则和示例
