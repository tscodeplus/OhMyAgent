v0.8.1 brings dynamic model resolution for built-in providers, sqlite-vec support on Termux (android-arm64), graceful OAuth token refresh, and restart notifications in WebUI.

## 🚀 New Features

### Dynamic Model Resolution for Built-in Providers

Built-in providers now expose their full model catalog dynamically:

- New `/api/providers` endpoint returns the live provider list from pi-mono, replacing the old hardcoded `BUILTIN_PROVIDERS` registry
- Any model ID supported by the API gateway (NVIDIA, etc.) can be used without manual registration — the resolver clones the first model of the matching provider to inherit headers, baseUrl, and compat settings, then substitutes the requested model ID
- Provider settings UI now shows each built-in provider's default baseUrl, making it clear which endpoint will be used

### sqlite-vec on Termux (android-arm64)

- Pre-compiled sqlite-vec v0.1.9 binary for Android aarch64 (Bionic libc, API 24+), built with Termux clang/NDK r29
- On Termux, falls back to the bundled native binary when the npm prebuilt is unavailable
- `dbsWithFailedExtension` guard prevents repeated load attempts and console noise on unsupported platforms

## 🛠 Improvements

- **Graceful OAuth token refresh** — `getOAuthApiKey` returns null instead of throwing on refresh failure, preventing cascading auth errors
- **Smart restart notification** — config reload now detects which sections actually changed; WebUI only prompts restart when channels, embedding, or database config is modified
- **WebUI restart toast** — config change broadcasts via WebSocket, and `AppShell` renders an in-app notification instead of silently applying changes
- **Provider default baseUrl in UI** — built-in provider expanded view falls back to the default baseUrl when no custom override is set; "Add Provider" modal shows it as read-only

## 🐛 Bug Fixes

- **NVIDIA model reasoning** — dynamic model clones from `getModel` inherit `reasoning: false` from the first NVIDIA template; added a runtime patch to override reasoning from `customProviders` config and enable `supportsReasoningEffort` in compat when reasoning is true
- **Header cleanup** — strip `NVCF-POLL-SECONDS` header from resolved models to avoid async inference timeouts on models that don't support it (e.g. MinimaxAI)
- **pnpm 11 compatibility** — moved `onlyBuiltDependencies` from `package.json` (no longer read by pnpm 11) to `.npmrc`

v0.8.1 带来内置 Provider 动态模型解析、Termux sqlite-vec 支持（android-arm64）、OAuth Token 优雅刷新和 WebUI 重启通知。

## 🚀 新功能

### 内置 Provider 动态模型解析

内置 Provider 现在动态暴露完整模型目录：

- 新增 `/api/providers` 端点，返回 pi-mono 实时 Provider 列表，替代旧硬编码的 `BUILTIN_PROVIDERS` 注册表
- API 网关支持的任意模型 ID（如 NVIDIA 等）无需手动注册即可使用 — 解析器克隆匹配 Provider 的第一个模型以继承 headers、baseUrl 和 compat，然后替换为请求的模型 ID
- Provider 设置界面现在显示每个内置 Provider 的默认 baseUrl，清楚展示实际使用的端点

### Termux 支持 sqlite-vec（android-arm64）

- 预编译 sqlite-vec v0.1.9 二进制文件，面向 Android aarch64（Bionic libc, API 24+），使用 Termux clang/NDK r29 构建
- Termux 上 npm 预编译不可用时自动回退到内置 native 二进制
- `dbsWithFailedExtension` 守卫防止不支持的平台上重复加载和日志刷屏

## 🛠 改进

- **OAuth Token 优雅刷新** — `getOAuthApiKey` 刷新失败时返回 null 而非抛异常，避免级联认证错误
- **智能重启通知** — 配置重载时检测哪些部分实际发生变更；仅当 channels、embedding 或 database 配置变更时提示重启
- **WebUI 重启提示** — 配置变更通过 WebSocket 广播，`AppShell` 渲染应用内通知，不再静默应用更改
- **Provider 默认 baseUrl 展示** — 内置 Provider 未设置自定义地址时显示默认 baseUrl；「添加 Provider」弹窗以只读方式展示

## 🐛 修复

- **NVIDIA 模型 reasoning** — `getModel` 动态克隆时继承了第一个 NVIDIA 模板的 `reasoning: false`；新增运行时补丁从 `customProviders` 配置覆盖 reasoning，并在 reasoning 为 true 时启用 `supportsReasoningEffort`
- **Header 清理** — 从已解析模型中剥离 `NVCF-POLL-SECONDS` header，避免不支持该特性的模型（如 MinimaxAI）出现异步推理超时
- **pnpm 11 兼容** — 将 `onlyBuiltDependencies` 从 `package.json`（pnpm 11 不再读取该字段）迁移至 `.npmrc`
