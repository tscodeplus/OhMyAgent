v0.7.0 is a major release focused on memory system overhaul, complete i18n of user-facing strings, English-only LLM prompts, a redesigned nightly maintenance orchestrator, and QQ channel reliability fixes.

## 🚀 New Features

### Memory System Overhaul

- **jieba Chinese word segmentation for FTS5** — pre-tokenize Chinese text before full-text indexing via unicode61; query-side jieba tokenization with OR-joined MATCH terms; standalone `memories_fts_jieba` table with auto-backfill on first startup and real-time sync
- **Embedding auto re-index** — detect provider/model/dimension changes across restarts; auto-drop old vectors and mark `needsReindex` on config change
- **Memory quality gate** — filter pure symbols, code blocks, and high-noise content before LLM extraction via `shouldExtractL1()`
- **L0→L3 formal pipeline** — explicit data flow: L0 conversation → L1 atomic → L2 scene → L3 persona, with per-layer status and error collection
- **Enhanced L3 persona** — new dimensions: `workHabits`, `knowledgeDomains`, `projectPreferences`
- **TTL cleanup safety threshold** — refuse to delete >80% of temp-kind memories in a single pass to prevent accidental data loss

### Redesigned DreamCycle Nightly Orchestrator

The old DreamCycle was rewritten as a 7-phase nightly maintenance pipeline:

- **7 phases**: synthesize → backlinks → extract → sceneCluster → hygiene → embed → purge
- **Timezone-aware scheduling** via `Intl.DateTimeFormat` (default 2:00 AM system local)
- **Grace window**: skip heavy phases if started >120 minutes past scheduled time
- **Per-phase timeout** (30min default) with `AbortController` and isolated failure handling
- **Audit trail**: phase results written to `maintenance_runs` table
- **Memory merge synthesis**: sample random memories, KNN search, LLM merge for pairs with similarity ≥0.85
- **WebUI settings** section under Settings > Memory System with i18n (zh-CN/en)
- **Migration**: `memory_hygiene` and `scene_cluster` moved from MaintenanceScheduler to DreamCycle

## 🔧 Refactoring

### English-Only LLM Prompts

All LLM-facing prompts are now hardcoded in English. The prompts i18n system has been removed — `prompts.json` deleted from both locales. A response language instruction (`IMPORTANT: You MUST respond in {language}`) is injected at the base system prompt layer, derived from `config.memory.outputLanguage` or `uiLanguage`. All previously hardcoded Chinese prompts in persona-distiller, mermaid-phase-tagger, turn-counter, compress, vision-bridge, scheduler-services, and skill SKILL.md files have been translated to English.

### Complete User-Facing i18n

The last remaining hardcoded Chinese strings across the codebase have been converted to i18n:

- QQ channel dispatcher and approval handler
- Feishu media tool errors
- WeChat LLM markers (Chinese → English)
- Skill activation labels across all channels and frontend
- Error messages, approval prompts, media errors, UI toast and file labels

## 🐛 Bug Fixes

- **QQ channel rate-limit retry** — QQ Bot API returns HTTP 400 (err_code 40023001) instead of standard 429 on rate limits. During deploy restarts, multiple rapid startup attempts would leave the QQ channel permanently offline. Added `fetchWithRetry()` with exponential backoff (2s→32s, up to 5 retries) covering both `getAccessToken()` and `getGatewayUrl()`.

## 🚀 新功能

### 记忆系统全面升级

- **jieba 中文分词 FTS5** — 中文文本在全文索引前自动分词（unicode61）；查询端 jieba 分词 + OR 组合 MATCH 词条；独立 `memories_fts_jieba` 表，首次启动自动回填、实时同步
- **Embedding 自动重建索引** — 检测 provider/model/dimension 变更；配置变化时自动清除旧向量并标记 `needsReindex`
- **记忆质量门禁** — LLM 提取前过滤纯符号、代码块和高噪声内容（`shouldExtractL1()`）
- **L0→L3 形式化管道** — 明确数据流：L0 对话 → L1 原子 → L2 场景 → L3 画像，附层级状态和错误收集
- **增强 L3 画像维度** — 新增 `workHabits`、`knowledgeDomains`、`projectPreferences`
- **TTL 清理安全阈值** — 单次清理拒绝删除超过 80% 的临时记忆，防止意外数据丢失

### 重新设计的 DreamCycle 夜间编排器

旧 DreamCycle 重写为 7 阶段夜间维护管道：

- **7 个阶段**：synthesize → backlinks → extract → sceneCluster → hygiene → embed → purge
- **时区感知调度** — `Intl.DateTimeFormat`（默认系统本地时间凌晨 2:00）
- **宽限期机制** — 超过预定时间 120 分钟后跳过重阶段
- **每阶段超时**（默认 30 分钟）+ `AbortController` + 独立故障处理
- **审计追踪** — 阶段结果写入 `maintenance_runs` 表
- **记忆合并合成** — 随机采样、KNN 检索、相似度 ≥0.85 的记忆对 LLM 合并
- **WebUI 设置** — Settings > 记忆系统，支持中英文
- **迁移** — `memory_hygiene` 和 `scene_cluster` 从 MaintenanceScheduler 迁移到 DreamCycle

## 🔧 重构

### LLM 提示词全面英文化

所有 LLM 交互的提示词现已硬编码为英文。提示词 i18n 系统已移除 — 两个语言目录的 `prompts.json` 已删除。回答语言指令（`IMPORTANT: You MUST respond in {language}`）注入到基础系统提示层，从 `config.memory.outputLanguage` 或 `uiLanguage` 派生。此前硬编码的中文提示词（persona-distiller、mermaid-phase-tagger、turn-counter、compress、vision-bridge、scheduler-services 及技能 SKILL.md）均已翻译为英文。

### 用户界面完整国际化

代码库中最后一批硬编码中文字符串已转为 i18n：

- QQ 频道调度器和审批处理器
- 飞书媒体工具错误信息
- 微信 LLM 标记（中文 → 英文）
- 全频道和前端技能激活标签
- 错误消息、审批提示、媒体错误、UI toast 和文件标签

## 🐛 修复

- **QQ 频道限流重试** — QQ Bot API 限流时返回 HTTP 400（err_code 40023001）而非标准 429。部署重启时多次快速启动会导致 QQ 频道永久离线。新增 `fetchWithRetry()` 指数退避重试（2s→32s，最多 5 次），覆盖 `getAccessToken()` 和 `getGatewayUrl()`。
