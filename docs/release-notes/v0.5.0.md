v0.5.0 delivers a major codebase restructuring alongside skill activation visibility across all channels, WebUI display fixes for tool and skill calls, and an upgrade of the embedded pi-mono framework to v0.79.6.

## 🚀 New Features

### Skill Activation Notifications

When a skill is triggered, you now see it clearly: a `⚡️ 技能激活: <name>` badge appears inline at activation time, rendered as a compact card in the message stream. This works across **all 5 channels** — WebUI, Feishu, Telegram, QQ, and WeChat.

- **Database persistence** — activation info is stored in message metadata, so page refreshes and app restarts don't lose it
- **showSkillCalls config** — toggle skill activation display on/off at the top level (config.yaml, .env, or WebUI Settings → General)
- **Card-style rendering** — activation cards use the same visual style as tool call cards for a consistent look

### pi-mono v0.79.6

The embedded AI agent framework is upgraded from v0.79.1 to v0.79.6:
- New `provider-env` module for environment-aware provider configuration
- Anthropic prompt cache write cost updated (2× input pricing for cache writes)
- Agent loop stability improvements (acceptingUpdates guard)

## 🛠 Improvements

### WebUI Config Fixes

- **showToolCalls** — the setting in Settings → General now actually hides/shows tool call cards in the WebUI, not just the toggle itself
- **showSkillCalls** — same setting properly persisted and respected across sessions

### Codebase

- **Bootstrap refactoring** — `bootstrap.ts` reduced from ~1,200 to 677 lines. Extracted 5 independent composer modules: agent-services, channel-services, memory-services, extension-services, and webui-services
- **Agent factory extraction** — `ModelResolver` and `ToolPipeline` split from `agent-factory.ts` into focused modules, each independently testable
- **Memory constructor cleanup** — `MemoryRetriever` and `MemoryWriter` switched to options-object constructors for cleaner dependency injection
- **Hot-reload consolidation** — all config hot-reload handlers moved into their respective composer modules
- **Code quality** — removed redundant `as any` casts, added debug logs to empty catch blocks, applied 18-file code review pass

## 🚀 新功能

### 技能激活通知

技能触发时，你现在可以清晰看到：`⚡️ 技能激活: <名称>` 标记在激活时刻内联出现，以紧凑卡片形式渲染在消息流中。支持**全部 5 个渠道**——WebUI、飞书、Telegram、QQ 和微信。

- **数据库持久化** — 激活信息存储在消息元数据中，页面刷新或应用重启不会丢失
- **showSkillCalls 配置** — 顶级开关控制技能激活显示的启用/禁用（config.yaml、.env 或 WebUI 设置 → 通用）
- **卡片式渲染** — 激活卡片使用与工具调用卡片相同的视觉风格，保持一致的外观

### pi-mono v0.79.6

嵌入式 AI Agent 框架从 v0.79.1 升级至 v0.79.6：
- 新增 `provider-env` 模块，支持环境感知的供应商配置
- Anthropic 提示缓存写入成本更新（缓存写入按输入价格的 2 倍计费）
- Agent 循环稳定性改进（acceptingUpdates 保护）
