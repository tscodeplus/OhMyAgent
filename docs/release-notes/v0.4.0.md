v0.4.0 is a major upgrade to the Skills system — community skill marketplace, structured SKILL.md authoring format, lifecycle tools, compliance tracking, and metrics-driven evolution. Every part of the skill lifecycle, from discovery to creation to monitoring, is now supported.

## 🚀 New Features

### Skill Marketplace

Browse, search, and install community skills from the WebUI and desktop app. Integrates with [skills.sh](https://skills.sh) (trending / popular / search) and [skillhub.cn](https://skillhub.cn) (Chinese-language skills). One-click install via npm/npx or zip download.

### Structured SKILL.md Format

New priority-aware body sections: `MUST` (hard constraints), `SHOULD` (guidelines), `WHEN` (triggers), `Checklist` (verification), and `Examples` (few-shot pairs). Each layer is compiled with the correct priority by the skill compiler.

### Skill Lifecycle Tools

Three new Agent tools:
- `skill_create` — create new skills from 4 built-in templates (agent-role / best-practice / minimal / tool-wrapper)
- `skill_lint` — validate SKILL.md correctness against the structured format spec
- `skill_test` — run a skill against a test message and report compliance + quality results

### Skill Compliance System

Session-scoped violation counting with automatic system prompt reinforcement. The agent is reminded of MUST rules it has violated, with escalating urgency as violations accumulate.

### Skill Metrics & Evolution

Background satisfaction inference (tracking user edits / re-runs / abandons), improvement proposal generator that suggests trigger/tool/prompt optimizations, and health diagnostic endpoints in the WebUI.

### Skill Review Agent

Built-in `skill-review` persona that runs health diagnostics across all installed skills, detecting stale triggers, unused tools, and compliance drift.

### CLI Commands

New `ohmyagent skill` subcommands: `lint`, `create`, `test`, and `list`. Manage skills from the terminal without the WebUI.

## 🛠 Improvements

### Skills System

- **Skill prompt moved to system prompt** — Activations no longer injected as `<system-reminder>` blocks in user messages, improving Anthropic prompt cache hit rate by 60-70%
- **Skill ID now uses directory basename** — cleaner, more predictable skill identifiers
- **Auto-reload** — skill registry automatically reloads when `file_write` creates or updates a SKILL.md
- **YAML list support** — `allowed-tools` field now accepts YAML list format in addition to comma-separated strings
- **Negation-aware compliance** — compliance parser skips negated rules (e.g. "do NOT do X") when extracting MUST requirements
- **Toast-based delete confirmation** — replaced `window.confirm` with the app's native toast for skill deletion
- **Marketplace integrated as sub-tab** — marketplace lives inside the Skills page, not a separate screen

### WebUI

- **Eager message persistence** — user messages are now persisted to the database *before* agent execution begins, preventing data loss on crash or navigation
- **Cross-session message isolation** — fixed a bug where messages could leak between sessions opened in different tabs
- **Agent completion push** — the agent now pushes a completion signal so the UI always knows when a response is fully finished

## 🚀 新功能

### Skill 市场

通过 WebUI 和桌面端浏览、搜索和安装社区技能。集成了 [skills.sh](https://skills.sh)（热门/流行/搜索）和 [skillhub.cn](https://skillhub.cn)（中文技能）。支持通过 npm/npx 或 zip 下载一键安装。

### 结构化 SKILL.md 格式

新增按优先级分层的正文区块：`MUST`（硬约束）、`SHOULD`（指导建议）、`WHEN`（触发条件）、`Checklist`（验证清单）和 `Examples`（示例对照）。每层由技能编译器按正确优先级编译。

### 技能生命周期工具

三个新的 Agent 工具：
- `skill_create` — 基于 4 个内置模板创建新技能（agent-role / best-practice / minimal / tool-wrapper）
- `skill_lint` — 按结构化格式规范验证 SKILL.md 的正确性
- `skill_test` — 用测试消息运行技能并报告合规性和质量结果

### 技能合规系统

会话级违规计数，自动强化系统提示。Agent 会收到其违反的 MUST 规则的提醒，违规累积后升级警告。

### 技能指标与演进

后台满意度推断（追踪用户编辑/重试/放弃）、改进建议生成器（建议触发词/工具/提示词优化）、WebUI 中的健康诊断端点。

### Skill Review Agent

内置 `skill-review` 角色，对所有已安装技能运行健康诊断，检测过期触发器、未使用工具和合规性偏差。

### CLI 命令

新增 `ohmyagent skill` 子命令：`lint`、`create`、`test` 和 `list`。无需 WebUI 即可在终端管理技能。

## 🛠 改进

### 技能系统

- **技能提示移至系统提示** — 技能激活不再作为 `<system-reminder>` 块注入用户消息，将 Anthropic 提示缓存命中率提高 60-70%
- **技能 ID 使用目录名** — 更清晰可预测的技能标识
- **自动重载** — `file_write` 创建或更新 SKILL.md 时自动重载技能注册表
- **YAML 列表支持** — `allowed-tools` 字段现在支持 YAML 列表格式，兼容逗号分隔字符串
- **否定感知合规解析** — 合规解析器提取 MUST 要求时跳过否定规则
- **Toast 确认删除** — 技能删除改用应用原生 toast 替代 `window.confirm`
- **市场作为子标签页** — 市场位于技能页面内，不再是独立页面

### WebUI

- **即时消息持久化** — 用户消息现在在 Agent 执行前就已存入数据库，防止崩溃或导航时的数据丢失
- **跨会话消息隔离** — 修复了不同标签页的会话之间消息可能泄漏的问题
- **Agent 完成推送** — Agent 完成时推送结束信号，UI 始终知道响应何时完全结束
