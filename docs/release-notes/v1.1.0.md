v1.1.0 introduces interactive `ask_user_question` — the agent can now ask you clarifying questions across all channels — plus a pi-mono engine upgrade (v0.80.2→v0.80.3), major CLI improvements, and install script hardening.

## ❓ Interactive Ask User Question

The agent can now pause mid-task and ask you questions via the `ask_user_question` tool. Supported across all channels:

- **Feishu** — interactive card with option buttons; free-text answers detected automatically
- **Telegram** — inline keyboard with callback queries; text answers with abbreviated callback data (fits 64-byte limit)
- **QQ** — keyboard buttons with `INTERACTION_CREATE` handler
- **WeChat** — text-based numbered options with `/answer` command
- **WebUI** — inline `UserQuestionCard` component with SSE events; auto-clears after answered; DB-persisted state
- **Commands** — `/answer` for manual text input; `/stop` `/clear` `/new` reject pending questions

Key behaviors: duplicate replies prevented, redundant ack messages removed, vertical button layout for all channels, human-readable answer reporting.

## 🧠 pi-mono v0.80.2 → v0.80.3

- **New models** — Claude Sonnet 5 models added with updated provider pricing across Anthropic, Amazon Bedrock, OpenRouter, and more
- **New utilities** — `error-body.ts` (HTTP error response parsing), `estimate.ts` (context token estimation), `retry.ts` (smart retry logic with backoff)
- **Usage.reasoning field** — new field for provider-reported thinking token breakdown
- **Bug fixes** — HTTP error response bodies now preserved, `streamSimple` max-token fix, undici crash fix, OpenAI Responses reasoning replay fix, Xiaomi MiMo pricing updates
- **Local patches** — `calculateContextTokens` guards against undefined usage from DB-restored history messages; all `.ts` import extensions corrected to `.js` for ESM compatibility
- **2259 tests passing**, 0 failures

## 🖥️ CLI Improvements

- **Port conflict prevention** — `ohmyagent` now detects when the service is already running before starting
- **Health-based startup detection** — uses the health endpoint instead of port binding to confirm the service is ready
- **launchd support (macOS)** — `ohmyagent stop` unloads the plist before killing the process to prevent `KeepAlive` respawn; `ohmyagent start` detects plist files and increases startup timeout
- **Global install** — CLI is now installed globally during setup via a wrapper script (not symlink), compatible with pnpm workspaces
- **curl|bash compatibility** — all `read` prompts redirected to `/dev/tty` for piped installs

## 🔧 Install Script Hardening

- **Non-interactive mode** — `install.sh` detects non-interactive sessions and skips prompts; config can be set via env vars and CLI args
- **esbuild builds approved** — ships `ui/pnpm-workspace.yaml` with `allowBuilds[esbuild]=true` to prevent `ERR_PNPM_IGNORED_BUILDS`; treated as non-network error in retry logic
- **System CA certificates** — sets `NODE_EXTRA_CA_CERTS` to system CA bundle on macOS and Linux
- **Proxy detection** — improved HTTP forward proxy verification with TUN fallback
- **WebUI token** — generated with mixed-case alphanumeric, matching Windows `install.ps1`

## 🎨 UI Polish

- **Native title bar** — reverted from `titleBarStyle: hidden` to native with a separator border to prevent header text overlap
- **Plan label i18n** — changed to 任务规划 / Task Plan with locale support
- **WebUI 404 fix** — corrected `uiRoot` path and `isDevMode` logic for production mode

## 🐛 Bug Fixes

- **set -e silent exit** — `grep -c` assignments now use `|| true` to prevent script termination
- **macOS proxy for update check** — uses system proxy when no env proxy is set
